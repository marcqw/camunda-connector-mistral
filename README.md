# Mistral AI Connector for Camunda 8

A curated [Element Template](https://docs.camunda.io/docs/components/modeler/desktop-modeler/element-templates/about-templates/) that brings **Mistral AI** into your Camunda 8 processes, chat completions, document OCR, embeddings, content moderation, classification, agents and conversations.

Built on top of the native `io.camunda:http-json:1` connector. No custom runtime needed.

![Camunda 8](https://img.shields.io/badge/Camunda-8.5%2B-blue) ![Mistral AI](https://img.shields.io/badge/Mistral-AI-orange) ![License](https://img.shields.io/badge/license-MIT-green)

## Features

- **18 operations** across 6 API domains, with human-readable labels
- Guided input fields for Chat completion: model picker, system prompt, user prompt, temperature, max tokens
- Pre-filled result expressions per operation so process variables are ready to use immediately
- `mistralUsage` automatically captured: prompt, completion and total tokens exposed as process variables
- Mistral API key stored as a Camunda secret(never hardcoded)
- Compatible with Camunda 8 SaaS and Self-Managed (8.5+)

## Token monitoring with Camunda Optimize

Every chat completion response includes a `mistralUsage` object that Camunda automatically stores as a process variable:

| Field               | Description                                                  |
| ------------------- | ------------------------------------------------------------ |
| `prompt_tokens`     | Tokens consumed by the system prompt + user prompt           |
| `completion_tokens` | Tokens generated by the model in its response                |
| `cached_tokens`     | Tokens served from Mistral's prompt cache (billed at reduced rate) |
| `total_tokens`      | Sum of prompt and completion tokens                          |

This matters in production. AI token consumption is a real operating cost, and anomalies (runaway prompts, unexpected spikes, model misbehaviour) are silent by default — you won't see them until the invoice arrives.

Because these values live as **process variables in Camunda**, [Optimize](https://docs.camunda.io/docs/components/optimize/what-is-optimize/) can turn them into operational intelligence:

- **Usage dashboards** — track `total_tokens` per process instance, per day, per business unit
- **Cost allocation** — correlate token consumption with specific process paths or user segments
- **Anomaly alerts** — trigger a notification when `total_tokens` exceeds a threshold (e.g. a bloated system prompt, a looping process, or a prompt injection attempt)
- **Efficiency tracking** — monitor `cached_tokens` ratio to measure prompt engineering improvements over time

## Installation

### Web Modeler

1. Open your project in [Web Modeler](https://modeler.camunda.io/)
2. Go to **Templates** → **Upload template**
3. Select `mistral-connector.json`

### Desktop Modeler

Copy `mistral-connector.json` into your element templates folder:

| OS      | Path                                                         |
| ------- | ------------------------------------------------------------ |
| Windows | `%APPDATA%\camunda-modeler\resources\element-templates`      |
| macOS   | `~/Library/Application Support/camunda-modeler/resources/element-templates` |
| Linux   | `~/.config/camunda-modeler/resources/element-templates`      |

Then restart the Modeler.

## Setup

### 1. Create a Mistral API key

Go to [console.mistral.ai/api-keys](https://console.mistral.ai/api-keys) and generate a key.

### 2. Store it as a Camunda secret

In [Camunda Console](https://console.camunda.io/) → your cluster → **Secrets**:

- **Name:** `MISTRAL_API_KEY`
- **Value:** your Mistral API key

See the [Camunda secrets documentation](https://docs.camunda.io/docs/components/console/manage-clusters/manage-secrets/) for details.

### 3. Use the connector in a process

1. Add a **Service Task** to your BPMN diagram
2. Click the wrench icon → **Select template** → pick **Mistral AI**
3. Choose an operation, fill in the input fields, configure the result expression

## Output mapping

Each operation ships with a pre-filled result expression. Examples:

**Chat completion**

```feel
= {
  mistralReply: response.body.choices[1].message.content,
  mistralUsage: response.body.usage
}
```

**OCR**

```feel
= {
  ocrPages: response.body.pages,
  ocrText: string join(for p in response.body.pages return p.markdown, "\n")
}
```

**Start / Append conversation**

```feel
= {
  conversationId: response.body.conversation_id,
  agentOutputs: response.body.outputs
}
```

Adjust the expressions in the **Output mapping** section of the connector to fit your process variables.

## Known limitations

- **File upload** (`POST /v1/files`) is not included — the `http-json` connector does not support `multipart/form-data`
- **Streaming** operations are not included — streaming responses are not compatible with synchronous job workers
- Fine-tuning, batch, workflows and observability endpoints are intentionally excluded to keep the template focused on process automation use cases
